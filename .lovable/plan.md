
# باني الصفحات المرئي الشامل

هدف: تقدر تفتح أي صفحة في الموقع (رئيسية، فئة، منتج، تشيك‌أوت، هيدر، فوتر) وتعدل عليها بالكامل — تسحب بلوكات من شريط جانبي وتدرجها مكان ما تبي، تعدل النصوص والألوان والأشكال للأزرار والبنرات وبطاقات المنتجات، كل شيء يتحفظ في قاعدة البيانات ويظهر للزوار.

## الشكل العام (UX)

```text
┌─────────────────────────────────────────────────────────────┐
│  [حفظ]  [نشر]  [تراجع]  [معاينة جوال/ديسكتوب]   عرض حي     │
├──────────┬──────────────────────────────────────────────────┤
│ بلوكات   │                                                  │
│ ────────│              الصفحة المباشرة                      │
│ نص      │         (تسحب من اليسار وتفلت هنا)               │
│ صورة    │                                                  │
│ بنر     │   كل بلوك عليه: [↕ سحب] [⚙ إعدادات] [🗑 حذف]    │
│ زر      │                                                  │
│ شبكة    │                                                  │
│ منتجات │                                                  │
│ تصنيفات│                                                  │
│ فاصل   ├──────────────────────────────────────────────────┤
│ HTML    │ لوحة الإعدادات (تفتح عند اختيار بلوك):           │
│ ...     │   المحتوى | الستايل | التجاوب | المتقدم          │
└──────────┴──────────────────────────────────────────────────┘
```

## المراحل

### المرحلة 1 — البنية التحتية وبلوكات أساسية (هذي الدفعة)

1. **جدول `page_blocks` في قاعدة البيانات**
   - `page_path` (مسار الصفحة أو نطاق عام مثل `global:header`، `global:footer`)
   - `parent_block_id` (للبلوكات المتداخلة لاحقاً)
   - `block_type` (نوع البلوك)
   - `position` (الترتيب)
   - `props` (jsonb — محتوى + ستايل)
   - `status` (draft / published)
   - حماية RLS: قراءة عامة للمنشور فقط، تعديل للأدمن فقط

2. **محرر مرئي جديد `PageBuilder`** يفتح بزر «تحرير الصفحة» من أي صفحة
   - شريط بلوكات جانبي قابل للسحب (drag from)
   - منطقة إفلات (drop zone) بين البلوكات وبعد آخر بلوك
   - مؤشرات بصرية لمواقع الإفلات
   - مكتبة @dnd-kit (موجودة فعلاً في المشروع)
   - حفظ تلقائي + زر «نشر»

3. **6 بلوكات أساسية للإطلاق:**
   - **نص** (heading / paragraph) — الخط/الحجم/اللون/المحاذاة
   - **صورة** — رفع أو URL، نسبة الأبعاد، الزوايا
   - **زر** — نص، رابط، شكل (مستطيل/مدور)، لون، حدود، ظل
   - **بنر** (صورة + نص فوقها + زر CTA)
   - **شبكة منتجات** — اختر تصنيف أو منتجات يدوياً + عدد الأعمدة + شكل البطاقة
   - **فاصل / مسافة** — ارتفاع متغير

4. **لوحة إعدادات لكل بلوك:**
   - تبويب «المحتوى»: نصوص، روابط، اختيار منتجات/تصنيفات
   - تبويب «الستايل»: ألوان، حشو، هامش، حدود، زوايا، ظل، خط
   - تبويب «التجاوب»: إخفاء/إظهار حسب الجوال أو الديسكتوب
   - كل التغييرات تنعكس فوراً في المعاينة

5. **عارض البلوكات `BlockRenderer`** — يقرأ من قاعدة البيانات ويرسم البلوكات في الصفحة الفعلية

### المرحلة 2 — تغطية باقي الموقع (الدفعة الثانية)

6. تحرير الهيدر والفوتر بنفس البلوكات (نطاق `global:header` و `global:footer`)
7. تخصيص شكل بطاقة المنتج (قالب يطبق على كل الكتالوج)
8. تخصيص صفحة المنتج (ترتيب الأقسام: صور، تفاصيل، مقاسات، أزرار، تقييمات)
9. تخصيص صفحة التشيك‌أوت (ترتيب الحقول، النصوص، شكل الأزرار)

### المرحلة 3 — بلوكات متقدمة وثيمات

10. بلوكات إضافية: كاروسيل، تابات، أكورديون، فيديو، خريطة، نموذج بريدي، عداد تنازلي، تقييمات
11. **ثيم الموقع**: لوحة ألوان عامة (أساسي/ثانوي/خلفية/نص) + خطوط، تنطبق على كل البلوكات تلقائياً
12. حفظ/استيراد قوالب جاهزة (Sections Library) — ينقر «أضف قسم» يختار من مكتبة قوالب

## تفاصيل تقنية

- **التخزين**: جدول `page_blocks` واحد + `props jsonb`. تاريخ التعديلات في `live_overrides_history` (موجود).
- **النشر**: عمود `status` — يفصل بين المسودة والنسخة المنشورة. زر «نشر» يحدث `status` و `published_at`.
- **المسارات**: المسار `page_path` يطابق مسار TanStack (مثلاً `/`، `/category/$slug`، `/product/$slug`). للنطاقات العامة نستخدم بادئة `global:`.
- **الأداء**: قراءة البلوكات من القاعدة في `loader` (TanStack Query `ensureQueryData`) عشان لا يومض الموقع.
- **الأمان**: زر «تحرير الصفحة» يطلع فقط للأدمن (`has_role`).

## ما لن يشمله هذا الباني (مبدئياً)

- لا يستبدل صفحات النظام الأساسية (السلة، الحساب، تتبع الطلب) — هذي تظل ثابتة
- لا يبني صفحات HTML من الصفر — يضيف بلوكات داخل قوالب التطبيق
- لا CSS مخصص يدوي — كل التحكم عبر اللوحة (يضمن عدم كسر التصميم)
